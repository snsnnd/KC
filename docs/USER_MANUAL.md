# 科创中心系统使用手册

## 系统概述

科创中心官网与运营系统是一套面向高校社团的轻量管理平台，涵盖公开官网、成员中心、管理员后台三大模块。系统基于 Node.js + Express 后端，前端为原生 HTML/CSS/JavaScript，数据以 JSON 文件持久化存储。

- **官网**：`https://47.120.0.45`
- **统一入口**：`https://47.120.0.45/portal.html`
- **成员中心**：`https://47.120.0.45/member.html`
- **管理后台**：`https://47.120.0.45/admin.html`

---

## 一、公开页面（无需登录）

### 1.1 官网首页 `/`

展示社团基本信息、项目展示和资源索引。所有访问者均可浏览。

### 1.2 资源中心 `/resources.html`

公开可用的资源列表，来自后台"资源链接"模块的配置。受权限保护的资源在此页面不会显示真实链接。

### 1.3 加入申请 `/join.html`

填写姓名、学号、班级、联系方式、申请部门和申请理由，提交加入社团的申请。系统会通知对应部门的管理员审核。

### 1.4 统一入口 `/portal.html`

管理员和成员的统一登录页面，顶部标签可切换"管理员登录"和"成员登录"。

### 1.5 账号激活 `/activate.html`

新成员首次登录前需要在此页面完成激活。输入管理员提供的成员账号、10 位一次性激活码，并设置自己的登录密码。激活成功后返回登录页手动登录。

### 1.6 邮件审批 `/email-approval.html`

管理员通过邮件中的一次性链接打开此页面，无需登录即可批准或拒绝材料和资金使用申请。

## 二、成员中心（需成员登录）

### 2.1 登录

在 `/member.html` 或 `/portal.html?type=member` 输入成员账号（如 `S2024130110`）和密码。首次使用前必须先完成激活。

### 2.2 导航栏

登录后左侧显示固定导航栏，包含四个功能模块，点击切换：

| 导航项 | 功能 |
|--------|------|
| 我的内部资源 | 查看有权访问的资源，获取链接和提取码 |
| 材料与资金申请 | 提交材料使用申请和资金使用申请 |
| 联系管理员 | 向负责部门的管理员发起问询 |
| 账号安全 | 修改当前密码 |

### 2.3 我的内部资源

- 显示当前权限集合下所有可访问的资源。
- 支持**搜索**（按名称和说明）和**类型筛选**。
- 合集资源显示为横跨两列的卡片，子资源嵌套在内。
- 点击"读取资源链接与凭据"获取真实链接地址和提取码。
- 每个资源可包含一个主链接和多个附加链接。

### 2.4 材料与资金申请

**材料申请**：选择材料、填写数量和用途后提交。提交后不会立即扣减库存，等待管理员审批。

**资金申请**：选择资金账户、填写金额和用途后提交。

**申请历史**：下方展示所有历史申请记录，包含审批状态。待审批申请可以撤销。

> 只有被授予 `material.request` 或 `fund.request` 权限的成员才能看到对应申请表单。

### 2.5 联系管理员

- 填写问询主题和说明后提交，系统自动通知负责成员所在部门的管理员。
- 管理员的回复会显示在问询线程中，发送到成员邮箱。
- 成员可以在原线程继续回复，已结束的问询会随新回复自动重新打开。

### 2.6 修改密码

输入当前密码和新密码（至少 8 位）后提交，密码修改后管理员后台同步更新。

## 三、管理后台（需管理员登录）

管理员在 `/portal.html` 选择"管理员登录"标签，使用管理员账号和密码登录。主管理员（owner）拥有全部权限；其他管理员根据分配的模块权限和部门范围看到不同内容。

### 3.1 工作区总览 `/admin.html`

登录后的默认页面，展示三个工作区的入口卡片。

### 3.2 运营宣发 `/admin.html?workspace=operations`

#### 3.2.1 基础信息

编辑社团名称、首屏标题和介绍、联系邮箱和申请通知邮箱。点击"保存更改"生效。

#### 3.2.2 项目内容

管理官网展示的项目卡片。每个项目包含：

- 系统编号、项目名称、分类
- 主题色、标签（逗号分隔）
- 简介
- 视频地址和海报地址（上传到 `/uploads/` 后填入）
- 最多 6 个附加链接（如演示地址、源码仓库）

支持按分类筛选和文本搜索。

#### 3.2.3 招新部门

管理可申请的部门列表。每个部门包含：

- 部门 ID（如 `software`）、名称、介绍
- 账号简写（如 `S`）：新成员账号 = 简写 + 学号
- "当前开放申请"开关：关闭后该部门不会出现在加入申请表的选择中

#### 3.2.4 资源链接

管理受保护和公开的资源，支持三级嵌套结构：

- **顶级资源**：每个资源可设置 ID、名称、类型、主链接、介绍、公开说明、权限标识、提取码
- **附加链接**：每个资源最多 12 个具名链接
- **子资源**：每个资源最多 20 个直接子资源，最多三级
- 子资源继承祖先合集的权限，也可拥有独立的权限标识和提取码

> 权限标识留空表示公开资源；填写后只有拥有该权限的成员才能访问。

#### 3.2.5 申请收件箱

查看和管理加入申请：

- 每张申请卡显示申请人信息、申请理由、状态和操作按钮
- **通过/拒绝**：审批决定会发送结果邮件给申请人
- **审批意见**：可选填理由
- **通知此申请人**：向单个申请人发送系统通知
- **转为成员**：只有已通过的申请可以转成员，系统自动生成成员账号和激活码
- 结果可在成员创建前修正

#### 3.2.6 邮件通知

配置 QQ SMTP 邮件服务：

- SMTP 授权码（加密存储）
- 发件名称、回复邮箱、默认通知邮箱组
- 申请通知负责人：按管理员身份精确投递
- 材料和资金审批结果邮件模板（支持变量占位符）
- 加入申请结果邮件模板（支持变量占位符）
- 发送测试邮件验证配置

#### 3.2.7 通知中心

向成员和申请人发送系统通知邮件：

- 选择发送对象：全体成员、未转成员申请人、全体管理员、按部门、按权限、指定成员/申请人、外部邮箱
- 编辑标题和正文后发送
- 收件地址按邮箱去重

#### 3.2.8 成员问询

查看成员发起的问询线程（仅限负责部门内），可以回复和结束问询。管理员回复会显示在成员中心并发送邮件。

#### 3.2.9 媒体上传

上传图片（JPG/PNG/WebP/AVIF）和视频（MP4/WebM），单文件最大 100MB。上传后获得站内地址，可填入项目或资源的媒体字段。

### 3.3 人员管理 `/admin.html?workspace=people`

#### 3.3.1 人员管理

- **创建成员**：填写姓名、学号、班级、联系方式、部门、权限（逗号分隔）。系统自动生成账号和激活码，只显示一次。
- **编辑成员**：修改资料和权限
- **重新签发激活码**：为尚未激活的成员生成新激活码
- **删除成员**

#### 3.3.2 管理员

- **创建管理员**：从已激活的成员中选择，分配角色（内容编辑/申请审核/主管理员）、可管理模块和负责部门
- **编辑管理员**：调整角色、模块权限和部门范围
- **删除管理员**：移除后台权限，成员账号保留

#### 3.3.3 操作日志

查看所有操作审计记录，包含时间、操作人、动作、目标对象和来源。

### 3.4 资源与资金 `/admin.html?workspace=assets`

#### 3.4.1 物资库存

- **创建材料**：名称、SKU、分类、单位、数量、成本、位置
- **归档/重新启用**：归档后不出现在成员申请选项中
- **入库**：调整库存数量并记录原因
- **删除（仅主管理员）**：删除材料记录，保留流水，剩余库存自动核销。有待审批申请时禁止删除。

#### 3.4.2 资金账户

- **创建账户**：名称、币种、初始余额
- **归档/重新启用**：归档后不出现在成员申请选项中
- **入账**：增加余额并记录原因
- **删除（仅主管理员）**：删除账户记录，保留流水，剩余余额自动核销。有待审批申请时禁止删除。

#### 3.4.3 使用审批

- 查看负责部门内的所有材料/资金使用申请
- **批准**：在串行锁内扣减库存或余额，发送结果邮件
- **拒绝**：附带审批意见
- 同一申请不会被重复审批

## 四、权限体系

### 4.1 管理员角色

| 角色 | 权限范围 |
|------|---------|
| 主管理员（owner） | 全部模块、全部部门、创建/删除管理员 |
| 内容编辑（editor） | 内容编辑、物资管理、成员管理、通知、审批 |
| 申请审核（reviewer） | 查看申请/审批/物资/资金，审批使用申请 |

### 4.2 面板权限

管理员可被授予的模块面板：`settings`、`projects`、`departments`、`resources`、`applications`、`mail`、`uploads`、`members`、`inventory`、`funds`、`usage`、`audit`。`notifications` 是所有管理员的基础能力。

### 4.3 部门范围

非主管理员只能查看和操作其负责部门内的数据（申请、成员、问询等）。跨部门对象返回 404 防止枚举。

### 4.4 成员权限

成员权限以逗号分隔的标识符表示，如 `resource.basic, material.request, fund.request`：

- `*`：通配符，拥有全部权限
- `resource.xxx`：对应资源的权限标识，拥有后才能访问
- `material.request`：材料申请权限
- `fund.request`：资金申请权限

## 五、安全特性

- **一次性激活码**：10 位随机字符，默认 7 天有效，服务器仅保存 HMAC 哈希
- **邮件审批 Token**：32 字节随机值，HMAC 哈希存储，24 小时过期，通过 URL Fragment 传递避免日志泄露
- **串行锁**：库存/资金审批在进程内排队，防止重复扣减
- **密码哈希**：成员密码使用 scrypt 加盐哈希；SMTP 授权码和资源提取码使用 AES-256-GCM 加密
- **申请限流**：每位申请人 30 分钟最多 3 次提交；每个 IP 10 分钟 30 次
- **CSRF 防护**：管理员写操作需携带 CSRF Token
- **缓存控制**：所有 API 响应设置 `Cache-Control: no-store`

## 六、部署概要

```
项目结构：
├── public/          # 静态页面（Nginx 直接提供）
├── backend/
│   ├── src/         # Express API
│   ├── config/      # 初始内容配置
│   └── tests/       # 端到端测试
├── deploy/
│   ├── nginx/       # Nginx 配置
│   ├── systemd/     # systemd 服务
│   └── scripts/     # 运维脚本
└── docs/            # 架构文档
```

**运行环境**：
- Nginx 提供静态文件 + HTTPS + 反向代理
- Node.js + Express 运行在 `127.0.0.1:3000`
- systemd 管理服务进程，以 `www-data` 用户运行
- 数据目录：`/var/lib/tech-club`（JSON 文件，权限 600）
- 上传目录：`/var/lib/tech-club/uploads`

**测试**：
```bash
cd backend
npm run test:applications    # 加入申请与激活码
npm run test:permissions     # 成员权限隔离
npm run test:resources       # 物资资金审批与生命周期
npm run test:admin-scope     # 管理员模块与部门隔离
npm run test:email-approval  # 邮件审批流程
npm run test:smoke           # 生产环境冒烟测试
```
